# Model card: Liu 2025 — optimal experiment design for identifiability & model discrimination

**Paper/thesis:** Y. Liu, P. K. Maini, R. E. Baker, "Optimal experiment design for practical parameter identifiability and model discrimination." arXiv:2506.11311v1 [q-bio.QM], 12 Jun 2025.
**Stage(s):** none (cross-cutting methods — inverse-problems / experiment-design family); serves parameterization of extraction, flow, bed_dynamics · **Kind:** calibration (offline methodology/tooling; never runtime)
**Status:** proposed

## Scope and mechanism
A methods paper on optimal experimental design (OED) for ODE models. It solves two distinct problems by designing a scalar control input u(t): (I) maximise **practical parameter identifiability** of one model with unknown parameters, measured by the width of the 95% profile-likelihood confidence interval; and (II) **model discrimination** between two fully-specified competing models, formulated as an optimal control problem and solved with Pontryagin's Maximum Principle (PMP) via a forward-backward sweep. All demonstrations use synthetic cell-proliferation data (logistic, Richards, and a linear signalling-pathway ODE); the biology is illustrative. The method itself is application-agnostic and maps onto two standing puckworks needs: (1) **identifiability auditing** of thin-data parameter fits we already carry, and (2) **OED for model discrimination** among the backlog's competing-hypothesis pairs, using MachineState P(t) as the control input u(t).

## Governing equations
Transcribed as the machinery that would actually be run, with the example ODEs kept only as context.

**General ODE + observation model (Eqs. 1–2):**
- dx/dt = f(x, t, u; θ), x(0) = x₀
- yᵢ(θ,u) = g(x(tᵢ; θ,u); θ) + ξᵢ, ξᵢ ~ N(0, σ²), i = 0…n_t

**Objective I — profile-likelihood identifiability**
- Normalised univariate profile likelihood (Eq. 3): lᵢ(θ′ᵢ) = max_{θ₋ᵢ,σ}[ log L(y_data | θ, σ) |_{θᵢ=θ′ᵢ} ] − log L(y_data | θ*, σ*)
- 95% CI region (Eq. 4): { θᵢ | lᵢ(θᵢ) > −χ²(0.95;1)/2 ≈ −1.92 }; interval width Δθᵢ is the quantity minimised over u.
- Control is a window function (Eq. 5): u(t) = u_max for τ₀ < t < τ₀+τ, else 0. Three control parameters (u_max, τ₀, τ).
- Cheap sensitivity heuristic (Eqs. 13–14): φ_θⱼ(t) = ∂C_model/∂θⱼ, solved by dφ_θⱼ/dt = (∂f/∂C)φ_θⱼ + ∂f/∂θⱼ, φ(0)=0; "sensitive interval" T_θⱼ = { t : |φ_θⱼ(t)| ≥ ½ max_t|φ_θⱼ(t)| }. Heuristic: place the control window to cover ~half of T_θⱼ. Authors show this is **only a heuristic** — it fails to detect that additive u_δ cannot restore identifiability (Sec. 2.3).

**Objective II — model discrimination (Eqs. 18–19, 27–28)**
- Cost functional: argmax_u J[u] = argmin_u ∫₀ᵀ L dt, with L = −[g₁(x₁(t;u)) − g₂(x₂(t;u))]² + α·u(t). α weights control cost; U bounds the control.
- Solved by PMP: adjoint dλᵢ/dt (Eq. 27), Hamiltonian H = −L + λ·f, pointwise update u_new = argmax_{u′∈U} H (Eq. 28), iterated by forward-backward sweep with adaptive step ω (Algorithm 1). Continuous vs "bang-bang" (two-valued) controls both considered.

**Example ODEs (illustrative only):** logistic dC/dt = rC(1−C/K) − δC (Eq. 6); Richards dC/dt = rC[1−(C/K)^γ] (Eq. 24). Controls u_K, u_δ, u_r modulate K, δ, r additively (Eqs. 11, 15, 16) or multiplicatively (Eq. 23).

## Calibration and validation offered by the source
No experimental validation of any kind. Every result is a self-consistent demonstration on **synthetic** data generated from the models' own assumed parameters. Structural identifiability claims (Supp. C) are analytic (Taylor-series method), not data-validated. The "ground-truth guess" values are stated to approximate a real MDCK proliferation fit from ref. [27], but that fit is not reproduced or checked here. In the registry's validation-strength vocabulary this is **verification/demonstration only — no independent experimental gate**. Authors flag their own key limitation: the design depends on fixed assumed parameter values, and requires knowing the control-to-parameter coupling (additive vs multiplicative) in advance, else it introduces new unknowns and nets no information.

## Assumptions and validity range
- ODE (lumped, deterministic) systems with Gaussian iid observation noise of known σ; fixed evenly-spaced observation grid (T, n_t, Δt not part of the design).
- A single scalar control, applied one parameter at a time; control mechanism (additive/multiplicative) assumed known.
- Requires a pre-specified "ground truth" parameter set to drive the design (authors call this the main limitation).
- **Silent on:** PDE/spatial models, stochastic dynamics, joint multi-parameter identifiability optimisation, priors over parameters (all listed as future work). Breaks when the control coupling is unknown, or when profile likelihoods are multi-modal (CI becomes a union of disjoint intervals — acknowledged).

## Interface mapping
Inputs consumed / outputs produced: **no contract fields** — this is not a process-model component. It runs offline, wrapped around an existing component plus a dataset. Two concrete couplings:
- **Identifiability audit:** feed a candidate model + a real/proposed shot dataset (ShotResultState traces); returns profile-likelihood CIs on that model's fitted parameters. Applies to Cameron's two-population rate constants and brewer2026's sigma(phi1) closure.
- **OED for discrimination:** feed two competing models sharing MachineState/BedState; treat MachineState P(t) as the control u(t); PMP returns the pressure profile that maximally separates their ShotResultState predictions.

Adapters/adaptation: nontrivial. The espresso flow/extraction stack is a coupled PDE/DAE system; the paper's PMP and profile-likelihood loops are demonstrated only on scalar ODEs (see failure modes).

## Implementation estimate
An offline methods capability, not a shot-chain component. Effort **M–L**: port the MATLAB FBS/PMP and profile-likelihood routines (published), then adapt from scalar ODEs to our coupled PDE/DAE flow+extraction models — the adaptation, not the port, is the cost, and the paper is explicitly silent on the PDE case. Dependencies: a differentiable/forward-solvable version of the target component and a defined observation model g(·) mapping states to ShotResultState observables. Natural gate: first application should be a bounded ODE-reducible sub-problem (e.g., identifiability of Cameron's rate constants on fixed traces) before attempting P(t) discrimination on the full stack.
