# docs/public — the public-value layer (PV-00)

This directory is the **generated** public face of `puckworks`: traceable claim
cards derived from the scientific harness, one per public story. It is the concrete
first build of the public-value track (`docs/PUBLIC_VALUE.md`, tasks PV-00–PV-18);
`docs/SPRINTS.md` "Public-value track (PV)" tracks status.

## What is here

- `generated/claims.json` · `generated/claims.csv` · `generated/claims.md` —
  produced by `python -m puckworks.public export`. **Do not hand-edit** — they are
  regenerated from the claim registry.

The claim registry itself is code, not prose:
`puckworks/public/` (`schema.py`, `claims.py`, `export.py`) — a **presentation-only**
package, NOT a registered physics component.

## The integrity guarantee (why this exists)

Every public number is **generated by a named function** (a `Producer` pointing at a
harness/model function), never hand-typed. Each claim also carries, structurally:

- a **unit** on every number;
- an **evidence-strength** label copied UNCHANGED from the scientific analysis
  (`measured/independent` · `post-fit reconstruction` · `verification` ·
  `qualitative` · `reference` · `negative validation`);
- an **observation/reconstruction/prediction/simulation badge**;
- the **source datasets** (which must exist in `puckworks/data/MANIFEST.csv`);
- a validity range, primary caveat (scope sentence), practical implication, and a
  one-line reproduction command;
- the **source commit** it was generated at.

`tests/test_public_claims.py` fails if any of these is missing, if a number is
hard-coded (no Producer), if a grinder-dial comparison lacks a non-portability
warning, or if a stored snapshot has drifted from the live harness result (a revised
scientific result must invalidate its stale card). These are fast structural checks;
they are **not** physics gates and are **not** in `run_all_gates()`.

The public track **consumes** scientific results; it never relaxes their strength
labels (PUBLIC_VALUE.md §3). Reference-strength and labelled-proxy results stay
labelled in public form.

## Regenerate

```bash
python -m puckworks.public export --out docs/public/generated        # fast claims
python -m puckworks.public export --out docs/public/generated --slow # + slow Producers
```

**Live interactives** (GitHub Pages, deployed from `docs/public/site/`):

- **PV-03 "The Cup Hides the Clock"** — <https://trbrewer.github.io/puckworks/flat-valley/>. Generated
  from the committed Paper A result bundle (`docs/figures/paper_a/results.json`) via
  `python -m puckworks.public.flat_valley export|verify` into a compact hash-bound snapshot
  (`puckworks/public/data/pv03_flat_valley.json`) and a static, framework-free interactive under
  `docs/public/site/flat-valley/`. Every headline number traces to a named bundle field; the snapshot
  records the source-bundle SHA-256 so `verify` fails on scientific drift.
- **PV-05 "More Physics Made It Worse"** — <https://trbrewer.github.io/puckworks/model-composition/>.
  Generated by named producer functions (`kappa_t_ladder`, `degeneracy_rmse`, `composition_residual`,
  `simulate`) via `python -m puckworks.public.model_composition export|verify` into
  `puckworks/public/data/pv05_model_composition.json` and a static interactive under
  `docs/public/site/model-composition/`. It shows why an extraction-only model that tracks the flow
  becomes *worse* than a constant when a swelling branch is added through one shared porosity state —
  rejecting one tested composition, not swelling in general. Bound by SHA-256 to the CC-BY Waszkiewicz
  trace.

**Implemented, awaiting live activation:**

- **PV-04 "How We Falsified Our Own Espresso Headline"** — producer `public.analysis_autopsy.pv04_values`.
  Generated from `puckworks.harness.result1_magnitude_comparison` via
  `python -m puckworks.public.analysis_autopsy export|verify` into the deterministic, hash-bound snapshot
  `puckworks/public/data/pv04_analysis_autopsy.json` and a static four-scene interactive under
  `docs/public/site/analysis-autopsy/`. It exposes the actual **replicate-level** extraction-yield points
  per grinder dial (not just cell means), each cell's own descriptive within-cell SD, and n per cell; the
  corrected cell means are ordered with the middle dial below the coarse dial (18.27 → 19.38 → 19.62 % EY)
  and the middle-minus-coarse (dial 1.7 − dial 2.0) contrast's Welch 95% interval excludes zero — no
  observed interior maximum. Evidence is **partitioned per panel** (corrected replicate data =
  OBSERVED/independent; rounded-coefficient/refit audit = RECONSTRUCTED/verification; model interior
  feature = EXPLORATORY_SIMULATION/qualitative; the superseded headline carries no current badge), so the
  global OBSERVED badge never covers the simulated or derived panels. The snapshot records the CC-BY
  Schmieder source SHA-256; only DERIVED replicate-level values are published, no raw source CSV. The
  live URL and README activation follow after Pages deployment and live-hash verification.
